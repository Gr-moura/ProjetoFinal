Construtor da classe \doxylink{class_central_de_jogos}{Central\+De\+Jogos}. 

Este construtor é responsável por inicializar a classe \doxylink{class_central_de_jogos}{Central\+De\+Jogos}, carregando os dados dos jogadores cadastrados a partir de um arquivo de texto. O arquivo deve estar localizado no caminho especificado ({\ttfamily data/\+Dados\+Jogadores\+Cadastrados.\+txt}) e deve seguir um formato específico para que os dados sejam lidos corretamente.

O arquivo de dados deve conter, para cada jogador, as seguintes informações em ordem\+:
\begin{DoxyEnumerate}
\item Nome do jogador (uma linha completa).
\item Apelido do jogador.
\item Número de vitórias no Jogo da Velha.
\item Número de derrotas no Jogo da Velha.
\item Número de empates no Jogo da Velha.
\item Número de vitórias no \doxylink{class_lig4}{Lig4}.
\item Número de derrotas no \doxylink{class_lig4}{Lig4}.
\item Número de empates no \doxylink{class_lig4}{Lig4}.
\item Número de vitórias no \doxylink{class_reversi}{Reversi}.
\item Número de derrotas no \doxylink{class_reversi}{Reversi}.
\item Número de empates no \doxylink{class_reversi}{Reversi}.
\item Número de vitórias no Batalha Naval.
\item Número de derrotas no Batalha Naval.
\item Número de empates no Batalha Naval.
\end{DoxyEnumerate}

Caso o arquivo não exista ou esteja corrompido, o construtor exibirá uma mensagem de erro e prosseguirá sem carregar os dados, iniciando o sistema com uma lista vazia de jogadores.

\begin{DoxyNote}{Observação}
O arquivo de dados deve estar no formato correto. Caso contrário, o sistema pode falhar ao ler os dados e exibir uma mensagem de erro indicando que o arquivo está corrompido.
\end{DoxyNote}
\begin{DoxyWarning}{Aviso}
Se o arquivo não puder ser aberto, o sistema exibirá uma mensagem de erro e não carregará os dados. Isso pode ocorrer se o arquivo não existir ou se houver problemas de permissão. 
\end{DoxyWarning}

Destrutor da classe \doxylink{class_central_de_jogos}{Central\+De\+Jogos}. 

Este destrutor é responsável por salvar os dados dos jogadores cadastrados em um arquivo de texto ({\ttfamily data/\+Dados\+Jogadores\+Cadastrados.\+txt}) antes de liberar a memória alocada para a classe. O arquivo é sobrescrito com as informações atualizadas dos jogadores, incluindo nome, apelido e estatísticas de cada jogo (vitórias, derrotas e empates).

O formato do arquivo é o seguinte para cada jogador\+:
\begin{DoxyEnumerate}
\item Nome do jogador (uma linha completa).
\item Apelido do jogador, seguido pelas estatísticas de cada jogo na ordem\+:
\begin{DoxyItemize}
\item Vitórias no Jogo da Velha.
\item Derrotas no Jogo da Velha.
\item Empates no Jogo da Velha.
\item Vitórias no \doxylink{class_lig4}{Lig4}.
\item Derrotas no \doxylink{class_lig4}{Lig4}.
\item Empates no \doxylink{class_lig4}{Lig4}.
\item Vitórias no \doxylink{class_reversi}{Reversi}.
\item Derrotas no \doxylink{class_reversi}{Reversi}.
\item Empates no \doxylink{class_reversi}{Reversi}.
\item Vitórias no Batalha Naval.
\item Derrotas no Batalha Naval.
\item Empates no Batalha Naval.
\end{DoxyItemize}
\end{DoxyEnumerate}

Caso o arquivo não possa ser aberto para escrita, o destrutor exibirá uma mensagem de erro e não salvará os dados. Isso pode ocorrer se o diretório {\ttfamily data} não existir ou se houver problemas de permissão.

\begin{DoxyNote}{Observação}
O arquivo é sobrescrito a cada execução do programa, garantindo que os dados estejam sempre atualizados.
\end{DoxyNote}
\begin{DoxyWarning}{Aviso}
Se o arquivo não puder ser aberto, os dados não serão salvos, e uma mensagem de erro será exibida. Isso pode resultar na perda de informações se o problema não for corrigido. 
\end{DoxyWarning}
